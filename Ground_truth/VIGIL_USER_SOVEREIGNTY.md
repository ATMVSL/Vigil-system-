# VIGIL User Sovereignty

> The user's identity is inviolable.

## Core Principle

Every user of VIGIL maintains complete sovereignty over their identity, data, and experience. VIGIL serves the user — it does not control, extract from, or profile them.

## Sovereignty Guarantees

- **Privacy-first** — Local-first storage, per-user engine isolation
- **No transcripts** — Voice is transcribed in real-time and immediately discarded
- **No telemetry** — No third-party analytics or tracking
- **Encrypted** — All data encrypted at rest and in transit
- **User API keys** — Users can provide their own OpenAI API key, maintaining control over AI access
- **Per-user Mirrors** — Each Mirror is sovereign to its user, no shared state

## Data Protection

- No cross-user data sharing or bleed
- Intel is identity-scrambled before any system use
- Users control their own data lifecycle
- NDA enforced at signup

## Role System & Access Control

```
Operator → Certified → Admin → Superadmin → Founder
```

- New users start as **Operator** (pending Founder approval)
- Progression through course completion and certification
- **Founder** has absolute system authority
- Role assignments managed exclusively through the Founder Mirror
