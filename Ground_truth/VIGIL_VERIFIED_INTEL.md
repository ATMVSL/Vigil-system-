# VIGIL Verified Intel

> Intel is identity-scrambled and verified. It allows VIGIL to learn and improve. It never rewrites Doctrine.

## Purpose

The Verified Intel system is the upward flow of the waterfall. It allows VIGIL to learn from Mirror interactions without compromising user sovereignty or Doctrine integrity.

## Intel Flow

```
Mirrors generate results
       │
       ▼
Intel is IDENTITY-SCRAMBLED
       │
       ▼
Intel is VERIFIED
       │
       ▼
Verified Intel improves VIGIL
       │
       ✗ NEVER rewrites Doctrine
```

## Rules

- All Intel must be **identity-scrambled** before it can be used
- Identity scrambling ensures **no user can be identified** from the Intel
- Intel must be **verified** — proven and tested before it changes system behavior
- Verified Intel may improve Mirror operation and system performance
- Verified Intel **MUST NEVER** rewrite Doctrine
- **No cross-user bleed** — one user's data must not affect another user's Mirror
- **No one-size-fits-all collapse** — Intel improvements must not homogenize the system
