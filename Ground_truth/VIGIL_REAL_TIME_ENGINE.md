# VIGIL Real-Time Engine

> Live cognitive evaluation. Knowing when to speak and when to just be present.

## Capabilities

The Real-Time Engine provides:

- **Live cognitive evaluation** — Continuous assessment of user state
- **Situational awareness** — Context beyond the current message
- **Pattern recognition** — Identifying behavioral patterns across sessions
- **Critical thinking** — Not just responding, but understanding

## What the Engine Evaluates

- What is **said**
- What is **not said**
- **Tone**
- **Movement** (behavioral patterns)
- **Cadence** (rhythm of engagement)
- **Silence** (knowing when to just be present)

## Key Principle

> Tokens and state machines are **tools only**. They do not determine truth, authority, or identity.

The Real-Time Engine uses AI models as tools to enable cognitive evaluation, but the models are not VIGIL. VIGIL is the architecture. The models serve the architecture, not the other way around.

## 13-Stage Cognitive Loop Pipeline

The Mirror processes each interaction through 13 stages:

1. Entry
2. Cardinal Axiom guard
3. Drift detection
4. Pattern recognition
5–12. (Intermediate processing stages)
13. Doctrine-constrained response

Each stage is governed by Doctrine and filtered through the Self-Filling Waterfall.
