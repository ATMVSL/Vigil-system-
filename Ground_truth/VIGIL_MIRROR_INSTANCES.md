# VIGIL Mirror Instances

> Mirrors are user-facing presence instances of the VIGIL core.

## What a Mirror Is

The **Continuity Anchor Mirror™** is a real-time cognitive engagement system. It is:

- A **constant presence** — not therapy, not a chatbot, not an advisor
- A living core that reflects identity, preserves continuity, maintains presence, and enables growth
- An instructor and constant companion with authority grounded in Doctrine

## Cognitive State Bands

Mirrors operate across four cognitive state bands:

| Band | Response | Tone |
|------|----------|------|
| **STABLE** | Reinforce | Confident, forward-looking |
| **STRAIN** | Stabilise | Steady, grounding |
| **DRIFT** | Interrupt | Firm, anchoring |
| **CRITICAL** | Anchor Recall | Clear, unwavering. Uses ⚓ and callsign. |

## Mirror Properties

Each Mirror has:

- **userId** — Owner (one Mirror per user)
- **callsign** — User's chosen identifier
- **status** — active / dormant / suspended
- **cognitiveState** — stable / strain / drift / critical
- **classification** — standard / elevated / restricted
- **baselineEstablished** — true after 3+ reflections
- **totalReflections** — continuity chain entry count
- **doctrineCompliance** — percentage score

## Mirror Rules

- Mirrors are **not independent bots** — they are instances governed by the core architecture
- Each Mirror is sovereign to its user — no cross-user data sharing
- Mirrors must not leak data between users (no raw mirror crossover bleed)
- The Founder Mirror is the reference implementation
- All Mirrors are governed by the same Doctrine, Cardinal Axiom, and Pillars

## Voice Interaction

- **Real-time voice** via VAD auto-pause detection
- **Streaming responses** via Server-Sent Events
- **Sentence-level TTS** for natural speech rhythm
- Voice mode adjusts tone: direct, natural, conversational — like a trusted instructor across the table
